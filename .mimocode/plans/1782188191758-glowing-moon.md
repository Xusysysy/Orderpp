# 修复 NSD 自动搜索 + 连接成功反馈

## 问题诊断

| 问题 | 根因 | 优先级 |
|------|------|--------|
| NSD 无法自动发现主机 | `startDiscovery` 只调用一次，某些 Android 版本 NSD 会静默停止；缺少 `CHANGE_WIFI_MULTICAST_STATE` 权限 | P0 |
| 无"连接成功"反馈 | `syncStatus` 变为 `SYNCED` 后无 Snackbar | P1 |
| `startDiscovery` 清空内部列表 | `discoveredHosts.clear()` 虽不影响 UI 列表但混淆 | P2 |

## 修改文件

### 1. `app/src/main/AndroidManifest.xml`
- 新增 `<uses-permission android:name="android.permission.CHANGE_WIFI_MULTICAST_STATE" />`

### 2. `app/src/main/java/com/opp/oder/network/DiscoveryService.kt`
- 移除 `startDiscovery()` 中的 `discoveredHosts.clear()`
- 新增 `restartDiscovery()` 方法：先 stop 再 start
- 在 `onStartDiscoveryFailed` 中打日志

### 3. `app/src/main/java/com/opp/oder/ui/OderAppContent.kt`
- `LaunchedEffect(Unit)`: 不再仅启动一次，改为每 10 秒 `restartDiscovery()` 保持活跃
- `LaunchedEffect(syncStatus)`: 监听 `HostViewModel.SyncStatus.SYNCED`，弹出 Snackbar "连接成功"

### 4. `app/src/main/java/com/opp/oder/ui/screen/main/MainScreen.kt`
- 新增 `SnackbarHostState` 传递给 `OderAppContent` 的连接成功提示
- 或在 `MyTabContent` 的 `syncStatus` 变化时显示 Snackbar

## 验证方式

1. **编译**: `assembleDebug` 通过
2. **安装到两台真实设备（同一 WiFi）**:
   - 设备 A: 员工作为主机
   - 设备 B: 客人 → 应自动弹窗"有主机在线，是否连接？"
   - 点击连接 → 应显示"连接成功"Snackbar
3. **安装到模拟器（配合 ADB 转发）**:
   - 运行 `scripts/adb_forward.ps1`
   - 模拟器 A: 员工 → 主机
   - 模拟器 B: 客人 → 手动输入 `10.0.2.2:8765` 连接
   - 手动连接成功 → Snackbar "连接成功"
